## 📝 КАК РАБОТИ ИЗЧИСЛЕНИЕТО (ПРОСТО ОБЯСНЕНИЕ)

### СТЪПКА 1: Събери отговорите

Потребителят отговаря на 60 въпроса по скала 1-5:
```
Въпрос 1 (R): Харесва ли ти да работиш с инструменти? → 5 (Силно ме интересува)
Въпрос 2 (I): Харесва ли ти да правиш експерименти? → 4 (Интересува ме)
Въпрос 3 (A): Харесва ли ти да рисуваш? → 2 (Не ме интересува особено)
...
Въпрос 60 (C): Харесва ли ти структура? → 5 (Силно ме интересува)
```

### СТЪПКА 2: Изчисли RAW SCORES

За всеки тип (R, I, A, S, E, C) събери отговорите на неговите 10 въпроса:

**Пример за тип R (Realistic):**
```
Въпрос 1 (R): 5
Въпрос 7 (R): 4
Въпрос 13 (R): 3

Въпрос 43 (R): 3
Въпрос 49 (R): 4
Въпрос 55 (R): 4
────────────────
R_raw = 5+4+3+5+2+4+5+3+4+4 = 39 точки
```

**Направи това за всички типове:**
```
R_raw = 39
I_raw = 47
A_raw = 24
S_raw = 18
E_raw = 22
C_raw = 40
```

### СТЪПКА 3: НОРМАЛИЗАЦИЯ (0-100 скала)

Минимум е 10 (10 въпроса × 1), максимум е 50 (10 въпроса × 5)

**Формула:**
```
normalized = ((raw - 10) / (50 - 10)) × 100
normalized = ((raw - 10) / 40) × 100
```

**Примери:**
```
R: ((39 - 10) / 40) × 100 = (29/40) × 100 = 72.5 ≈ 73%
I: ((47 - 10) / 40) × 100 = (37/40) × 100 = 92.5 ≈ 93%
A: ((24 - 10) / 40) × 100 = (14/40) × 100 = 35.0 = 35%
S: ((18 - 10) / 40) × 100 = (8/40) × 100 = 20.0 = 20%
E: ((22 - 10) / 40) × 100 = (12/40) × 100 = 30.0 = 30%
C: ((40 - 10) / 40) × 100 = (30/40) × 100 = 75.0 = 75%
```

**Краен резултат:**
```
R = 73%
I = 93%
A = 35%
S = 20%
E = 30%
C = 75%
```

### СТЪПКА 4: ГЕНЕРИРАЙ HOLLAND CODE

Вземи топ 3 типа по score:

```
1. I = 93%  ← Най-висок
2. C = 75%  ← Втори
3. R = 73%  ← Трети

Holland Code = "ICR"
```

## 💻 КОД ПРИМЕРИ

### JavaScript функция (минимална версия):

```javascript
function calculateRIASEC(answers) {
  // Mapping на въпросите
  const mapping = {
    R: [1, 7, 13, 19, 25, 31, 37, 43, 49, 55],
    I: [2, 8, 14, 20, 26, 32, 38, 44, 50, 56],
    A: [3, 9, 15, 21, 27, 33, 39, 45, 51, 57],
    S: [4, 10, 16, 22, 28, 34, 40, 46, 52, 58],
    E: [5, 11, 17, 23, 29, 35, 41, 47, 53, 59],
    C: [6, 12, 18, 24, 30, 36, 42, 48, 54, 60]
  };
  
  const scores = {};
  
  // Изчисли за всеки тип
  Object.keys(mapping).forEach(type => {
    // 1. Raw score (сумирай отговорите)
    const raw = mapping[type].reduce((sum, qId) => sum + answers[qId], 0);
    
    // 2. Normalize (0-100)
    scores[type] = Math.round(((raw - 10) / 40) * 100);
  });
  
  // 3. Holland Code (топ 3)
  const sorted = Object.entries(scores)
    .sort((a, b) => b[1] - a[1])
    .slice(0, 3);
  
  const hollandCode = sorted.map(([type, score]) => type).join('');
  
  return { scores, hollandCode };
}

// ИЗПОЛЗВАНЕ:
const answers = {
  1: 5, 2: 4, 3: 2, 4: 3, 5: 2, 6: 5,
  7: 4, 8: 5, 9: 1, 10: 2, 11: 3, 12: 4,
  // ... всички 60
};

const result = calculateRIASEC(answers);
console.log(result.scores);      // {R: 73, I: 93, A: 35, S: 20, E: 30, C: 75}
console.log(result.hollandCode); // "ICR"
```

## 📊 ИНТЕРПРЕТАЦИЯ НА SCORES

```
Score   Ниво            Значение
─────────────────────────────────────────────────
90-100  Много силен     Доминиращ интерес
70-89   Силен           Ясен интерес
50-69   Умерен          Известен интерес
30-49   Нисък           Слаб интерес
0-29    Много нисък     Липса на интерес
```

### Пример профил: R=73, I=93, A=35, S=20, E=30, C=75

**Интерпретация:**
- **I (93%)** - Много силен интерес към анализ, наука, изследвания
- **C (75%)** - Силен интерес към организация, структура, системи
- **R (73%)** - Силен интерес към практична, техническа работа
- **A (35%)** - Нисък интерес към творческа работа
- **E (30%)** - Нисък интерес към лидерство
- **S (20%)** - Много нисък интерес към работа с хора

**Holland Code: ICR**
→ Профил подходящ за: Data Science, Software Engineering, Research, Analytics

## 🔗 СВЪРЗВАНЕ С DATABASE

След като имаш `scores` и `hollandCode`, ги използваш за matching:

```sql
-- Намери подходящи специалности
SELECT 
    specialty_base_name,
    riasec_code,
    calculate_riasec_compatibility(
        '{"R": 73, "I": 93, "A": 35, "S": 20, "E": 30, "C": 75}'::jsonb,
        riasec_scores
    ) AS compatibility
FROM specialty_riasec_mapping
ORDER BY compatibility DESC
LIMIT 20;
```
