> **Документация за НОИТ 2024/2025**

## 1. Концепция, Визия и Проблем

### 1.1. Въведение и Визия
**"УниПът" (Uni-Path)** не е просто поредният информационен портал, а цялостна екосистема, която трансформира стресиращия процес на кандидатстване в структурирано, предвидимо и персонализирано преживяване. Визията ни е да демократизираме достъпа до информация за висше образование, като предоставим на всеки ученик – независимо от финансовите му възможности за частни консултанти – мощен инструмент за планиране на бъдещето.

### 1.2. Анализ на Проблема (The "Chaos" of Admissions)
В момента пътят на един кандидат-студент е изпълнен с препятствия:
*   **Информационна фрагментация:** Всеки от 50+ университета в България има собствен сайт с различна навигация и скрити PDF файлове с правила.
*   **Сложно балообразуване:** Един университет изисква матура по БЕЛ с коефициент 3, друг – с коефициент 1, а трети приема само с изпит. Учениците прекарват часове в ръчни сметки на хартия, често допускайки грешки.
*   **Липса на ориентация:** Много ученици избират специалност "на сляпо", без да знаят дали тя отговаря на техния личностен профил.
*   **Социално неравенство:** Учениците от по-малки населени места имат по-труден достъп до кариерни центрове и актуална информация.

### 1.3. Решението "УниПът"
Платформата обединява три критични стълба в едно SPA (Single Page Application):
1.  **Откриване (Discovery):** Интелигентна търсачка, която не просто филтрира по име, а позволява сравнение на условията.
2.  **Валидация (Calculation):** Уникален динамичен калкулатор, който "знае" правилата на всеки университет и автоматично адаптира интерфейса за въвеждане на оценки.
3.  **Насочване (Guidance):** Вграден кариерен съветник, базиран на научния модел RIASEC, който свързва личностните черти с конкретни академични програми.

### 1.4. Целева Аудитория
*   **Кандидат-студенти (11-12 клас):** Основна група. Търсят бърз и точен отговор на въпроса "Къде мога да вляза с моите оценки?".
*   **Родители:** Търсят сигурност и прозрачност за бъдещето на децата си, както и информация за общежития и битови условия.
*   **Училищни кариерни консултанти:** Използват платформата като помощен инструмент при работа с класове.

### 1.5. Потребителско Пътешествие (User Journey)
1.  *Фаза "Несигурност":* Потребителят влиза и прави RIASEC тест в секция **Кариерен Съветник**. Получава профил (напр. "Изследователски") и списък с подходящи професии.
2.  *Фаза "Проучване":* От професиите директно вижда свързаните специалности в секция **Университети**.
3.  *Фаза "Планиране":* Харесва "Софтуерно инженерство" и отива в **Калкулатор**. Системата му показва: "За тази специалност ти трябват ДЗИ БЕЛ и ДЗИ Математика".
4.  *Фаза "Реалност":* Въвежда прогнозните си оценки. Калкулаторът показва бал 28.50 и го сравнява с мин. бал от 2024 (27.80). Индикаторът свети в зелено.
5.  *Фаза "Логистика":* Проверява секция **Общежития**, за да види условията за живот в избрания град.

## 2. Технологична Архитектура

Проектът е реализиран като **Single Page Application (SPA)**, използвайки модерна React екосистема и Backend-as-a-Service (BaaS) архитектура.

| Компонент | Технология | Описание |
| :--- | :--- | :--- |
| **Frontend Framework** | [React 19](package.json) | Компонентно-базирана UI библиотека. |
| **Build Tool** | [Vite 6](vite.config.js) | Светкавичен build и HMR (Hot Module Replacement). |
| **Styling** | [Tailwind CSS 4](tailwind.config.js) + DaisyUI | Utility-first CSS с готови UI компоненти. |
| **Routing** | [React Router 7](src/app/App.jsx) | Клиентска навигация и защита на пътища. |
| **Backend / DB** | [Supabase](src/lib/supabase.js) | PostgreSQL, Auth, Realtime API. |
| **Icons & Maps** | Lucide React, Leaflet | Визуализация на икони и карти за общежития. |
| **Testing** | [Vitest](src/components/Calculator/GradeInputSection.test.jsx) | Unit тестове за критична логика (калкулатор). |

## 4. Потребителски Потоци (User Flows)

### 4.1. Автентикация и Сигурност
*   **Flow:** Потребителят влиза през `/login` или се регистрира.
*   **Механизъм:** Използва се Supabase Auth.
*   **Защита:**
    *   [`ProtectedRoute.jsx`](src/routes/ProtectedRoute.jsx): Проверява наличието на потребителска сесия. Ако няма -> редирект към Login.
    *   [`AdminRoute.jsx`](src/routes/AdminRoute.jsx): Проверява дали потребителят има роля/claim за администратор.

[SCREENSHOT: Екран за вход/регистрация | file: docs/images/login_screen.png]

### 4.2. Търсене на Университети
*   **Flow:** Потребителят отваря `/universities`. Може да филтрира по град или име/специалност.
*   **Логика:**
    1.  `universityService.searchUniversities` проверява локалния кеш (`localStorage`).
    2.  Ако няма данни -> заявка към Supabase (`university_admissions`).
    3.  Данните се филтрират клиентски (за бързодействие при малък обем данни).

### 4.3. Бал Калкулатор (Основна Функционалност)
*   **Flow:** `/calculator` (достъпен само за логнати).
*   **Стъпки:**
    1.  Избор на Факултет и Специалност.
    2.  Зареждане на **Коефициенти**: Системата извлича JSON конфигурация за избраната специалност.
    3.  **Въвеждане на Оценки:**
        *   Задължителни: БЕЛ (ДЗИ), Диплома.
        *   Динамични: Потребителят добавя предмети (до 5) чрез `GradeInputSection`.
    4.  **Валидация:** Проверка за диапазон 2.00 - 6.00.
    5.  **Изчисление:**
        *   Алгоритъмът избира най-високата оценка за даден слот (напр. Матура или Оценка от курс) според `SLOT_GROUPS`.
        *   Прилага множители от `coefficients`.

[SCREENSHOT: Калкулатор с въведени оценки и резултат | file: docs/images/calculator_demo.png]

### 4.4. Админ Панел
*   **Flow:** `/admin`.
*   **Функционалност:** Форма за добавяне на нови специалности.
*   **Data Entry:** Администраторът въвежда JSON обект за коефициентите директно (напр. `{"exam_mat": 3, "diploma": 1}`).
*   **Write Operation:** Запис в таблица `universities`.

## 5. Алгоритъм и Логика на Данните

### Структура на Коефициентите (JSON)
В базата данни (и локалния конфиг), правилата за балообразуване се пазят като JSON:

```json
{
  "dzi_bel": 1,        // Оценка от матура по БЕЛ * 1
  "math_group": 3,     // Най-добрата оценка от група "Математика" * 3
  "diploma": 1         // Общ успех от диплома * 1
}
```

### Псевдокод на Изчислението
Базиран на логиката в `Calculator.jsx`:

```javascript
function calculateScore(userGrades, coefficients) {
  let totalScore = 0;
  
  foreach (coeffKey, multiplier in coefficients) {
    // 1. Намиране на групата от предмети за този ключ (напр. math_group включва ДЗИ Математика, Изпит Математика и т.н.)
    let candidateGrades = SLOT_GROUPS[coeffKey] || [coeffKey];
    
    // 2. Намиране на най-високата валидна оценка от групата
    let maxGrade = 0;
    foreach (subject in candidateGrades) {
      if (userGrades[subject] > maxGrade) {
        maxGrade = userGrades[subject];
      }
    }
    
    // 3. Добавяне към бала
    totalScore += maxGrade * multiplier;
  }
  
  return totalScore;
}
```

## 6. Данни и Сигурност

### Схема на Данните (Supabase)
На база на кода (`universityService.js`, `AdminDashboard.jsx`), основните таблици са:

1.  **`university_admissions`** (Read-only за потребители):
    *   `id`, `university_name`, `city`, `specialty`, `faculty`
    *   `min_ball_2024` (float)
    *   `coefficients` (JSONB) - Ключово поле за калкулатора.

2.  **`dormitories`**:
    *   `id`, `name`, `university_id`, `price`, `conditions`, `location` (lat/lng).

### Сигурност
*   **RBAC (Role-Based Access Control):** Реализиран чрез проверка на потребителски роли/списъци (видим в `AdminRoute`).
*   **Client-Side Protection:** `ProtectedRoute` компонентът спира неоторизиран достъп до UI.
*   **Environment Variables:** API ключовете са изнесени (не са в кода), зареждат се чрез `import.meta.env`.

## 7. Инсталация и Пускане

Изисквания: Node.js 18+, npm/yarn.

1.  **Клониране и Инсталация:**
    ```bash
    git clone <repo-url>
    cd Uni-put
    npm install
    ```

2.  **Конфигурация:**
    Създайте `.env` файл в корена:
    ```env
    VITE_SUPABASE_URL=your_supabase_url
    VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
    ```

3.  **Стартиране:**
    ```bash
    npm run dev
    ```
    Приложението ще е достъпно на `http://localhost:5173`.

## 9. Тестване

Проектът използва **Vitest** и **@testing-library/react** за unit и интеграционни тестове.

### 9.1. Стартиране на тестовете

От корена на проекта:

```bash
npx vitest run
```

За измерване на покритие може да се използва `@vitest/coverage-v8`.  
Инсталация (по желание):

```bash
npm install -D @vitest/coverage-v8
```

След това:

```bash
npx vitest run --coverage
```

Критичните части за страницата „За нас“ и маршрута `/about` са проектирани да имат ≥ 90% тестово покритие чрез:

-   [`About.test.jsx`](src/components/common/About.test.jsx) – unit тестове за съдържанието и семантичната структура на страницата.
-   [`AppAboutRoute.test.jsx`](src/app/AppAboutRoute.test.jsx) – интеграционен тест, който валидира маршрута `/about` в React Router.
